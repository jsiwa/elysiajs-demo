📊 进化历程

❌ 第一版：硬编码链接

let productsLink = '/products' let loginLink = '/login' if (lang === 'ja') { productsLink = '/ja/products' loginLink = '/ja/login' } else if (lang === 'zh') { productsLink = '/zh/products' loginLink = '/zh/login' } <a href="${productsLink}">Products</a>

问题：重复代码，难以维护

✅ 最终版：直接传路径

import { createPathUtils } from '../lib/path-utils' const path = createPathUtils(lang) // 直接传入任何路径，自动处理多语言 <a href="${path.to('/products')}">Products</a> <a href="${path.to('/blog/hello-world')}">Blog Post</a> <a href="${path.to('/user/123/settings')}">User Settings</a>

优势：极简API，无限灵活

🚀 核心功能演示

1. 基础路径转换

const path = createPathUtils('zh') path.to('/') // → '/zh' path.to('/products') // → '/zh/products' path.to('/blog/hello') // → '/zh/blog/hello' path.to('/user/123') // → '/zh/user/123'

2. 带查询参数的路径

path.toWithQuery('/products', { category: 'electronics', page: '2' }) // → '/zh/products?category=electronics&page=2' path.toWithQuery('/search', { q: 'hello world', type: 'post' }) // → '/zh/search?q=hello+world&type=post'

3. 带锚点的路径

path.toWithHash('/docs', 'installation') // → '/zh/docs#installation' path.toWithHash('/blog/post-1', 'comments') // → '/zh/blog/post-1#comments'

4. 路径匹配和检查

path.matches('/zh/products', '/products') // → true path.startsWith('/zh/blog/hello', '/blog') // → true path.matches('/zh/admin', '/products') // → false

🔧 高级功能

1. URL 构建器 - 链式调用

import { createURL } from '../lib/path-utils' const url = createURL('zh', '/products') .query('category', 'electronics') .query('page', '2') .hash('reviews') .build() // → '/zh/products?category=electronics&page=2#reviews'

2. 路径模板 - 参数替换

import { pathTemplate } from '../lib/path-utils' pathTemplate('/user/:id/posts/:postId', { id: '123', postId: 'hello-world' }, 'zh' ) // → '/zh/user/123/posts/hello-world'

3. 批量路径处理

import { localizePaths, getAllLanguagePaths } from '../lib/path-utils' // 批量本地化 localizePaths(['/home', '/about', '/contact'], 'zh') // → ['/zh/home', '/zh/about', '/zh/contact'] // 生成所有语言版本 getAllLanguagePaths('/products') // → { en: '/products', ja: '/ja/products', zh: '/zh/products' }

4. 路径验证和标准化

import { PathValidator } from '../lib/path-utils' PathValidator.isValidPath('/products') // → true PathValidator.isAbsolutePath('/products') // → true PathValidator.isExternalLink('https://...') // → true PathValidator.normalizePath('//products///') // → '/products'

🌐 实际应用示例

在组件中使用

// Header.tsx import { createPathUtils } from '../lib/path-utils' export function Header({ lang }) { const path = createPathUtils(lang) return ` <nav> <a href="${path.to('/')}">Home</a> <a href="${path.to('/products')}">Products</a> <a href="${path.to('/blog')}">Blog</a> <a href="${path.to('/about')}">About</a> </nav> ` }

动态路径生成

// BlogPost.tsx function BlogPost({ lang, slug, category }) { const path = createPathUtils(lang) return ` <article> <a href="${path.to('/blog')}">← Back to Blog</a> <a href="${path.to(`/blog/category/${category}`)}">More in ${category}</a> <a href="${path.toWithHash(`/blog/${slug}`, 'comments')}">Comments</a> </article> ` }

表单和搜索

// SearchForm.tsx function SearchForm({ lang, query, filters }) { const path = createPathUtils(lang) const searchUrl = path.toWithQuery('/search', { q: query, category: filters.category, sort: filters.sort }) return `<form action="${searchUrl}">...</form>` }
